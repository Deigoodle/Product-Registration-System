# Product-Registration-System

Prueba de Diagnostico para proceso de selección de **Desis**

## Estructura

```
├── README.md
├── LEEME.txt
├── docker-compose.yml
├── api/
│   ├── Dockerfile
│   ├── app.php
│   ├── src/
│   │   ├── Controller/
│   │   ├── DB/
│   │   ├── Model/
│   │   ├── Service/
│   │   └── Validator/
├── sql/
│   ├── sample.sql
└── web/
    ├── index.html
    ├── css/
    └── js/
```

- `README.md` y `LEEME.txt`: Documentación del proyecto (Duplicado).
- `docker-compose.yml`: Configuración de Docker para la API y la base de datos.
- `api/`: Código fuente de la API REST.
  - `Dockerfile`: Instrucciones para construir la imagen de la API.
  - `app.php`: Punto de entrada de la aplicación.
  - `src/`: Código organizado por capas (Controladores, Modelos, Servicios, Validadores, etc.).
- `sql/`: Scripts SQL para crear la base de datos y tablas, y para insertar datos de ejemplo.
- `web/`: Archivos para la interfaz web (HTML, CSS, JavaScript).

## Requisitos

- PHP 8.0 o superior
- PostgreSQL 16 o superior (probablemente funcione con versiones anteriores).
- Docker y Docker Compose (opcional, para facilitar la configuración).

## Configuración

### Variables de Entorno y Credenciales (Opcional)

El proyecto ya viene configurado con valores por defecto que coinciden tanto con el entorno Docker como con una instalación local estándar de PostgreSQL. **No es obligatorio configurar variables de entorno para probar la aplicación.**

Sin embargo, si tu entorno local utiliza credenciales distintas, puedes modificarlas exportándolas en tu terminal antes de iniciar la aplicación o editando directamente el archivo `api/app.php`:

* `DB_DSN`: String de conexión (Por defecto: `pgsql:host=localhost;port=5432;dbname=product_db`)
* `DB_USER`: Usuario de la base de datos (Por defecto: `postgres`)
* `DB_PASS`: Contraseña de la base de datos (Por defecto: `adminpassword`)

### Configuración con Docker (Recomendada)

1. Clonar el repositorio.
    ```bash
    git clone https://github.com/Deigoodle/Product-Registration-System.git
    cd Product-Registration-System
    ```

2. Iniciar la base de datos y la API usando Docker Compose.
    ```bash
    docker-compose up -d
    ```

### Configuración manual (sin Docker)

Si prefieres no usar Docker, puedes levantar el entorno localmente siguiendo estos pasos:

1. Clonar el repositorio.
    ```bash
    git clone https://github.com/Deigoodle/Product-Registration-System.git
    cd Product-Registration-System
    ```

2. **Base de Datos:** Configura tu servidor local de PostgreSQL, crea la base de datos `product_db` y ejecuta el script de inicialización:
   ```bash
   psql -U postgres -c "CREATE DATABASE product_db;"
   psql -U postgres -d product_db -f sql/sample.sql
   ```

3. **API:** Ir a la carpeta `api/` y ejecutar la aplicación PHP:
   ```bash
   cd api
   php -S localhost:9999 app.php
   ```

## Uso
- La API estará disponible en `http://localhost:9999`.
- Abrir el archivo `web/index.html` en un navegador para acceder a la interfaz de usuario.

## Endpoints de la API

### Productos

- `GET /productos`: Listar todos los productos.
- `GET /productos/{id}`: Obtener detalles de un producto específico.
- `GET /productos/codigo/{codigo}`: Obtener detalles de un producto por su código.
- `POST /productos`: Registrar un nuevo producto (requiere JSON con los datos del producto).
- `PUT /productos/{id}`: Actualizar un producto existente (requiere JSON con los datos actualizados).
- `DELETE /productos/{id}`: Eliminar un producto.

### Monedas
- `GET /monedas`: Listar todas las monedas disponibles.
- `GET /monedas/{id}`: Obtener detalles de una moneda específica.

### Bodegas

- `GET /bodegas`: Listar todas las bodegas disponibles.
- `GET /bodegas/{id}`: Obtener detalles de una bodega específica.

### Sucursales

- `GET /sucursales`: Listar todas las sucursales disponibles.
- `GET /sucursales/{id}`: Obtener detalles de una sucursal específica
- `GET /sucursales/bodega/{bodega_id}`: Listar sucursales por bodega.

### Materiales

- `GET /materiales`: Listar todos los materiales disponibles.
- `GET /materiales/{id}`: Obtener detalles de un material específico.
